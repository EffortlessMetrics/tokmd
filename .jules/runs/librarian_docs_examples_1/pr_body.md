## 💡 Summary
Added practical examples to the CLI help text for `diff`, `gate`, `run`, `export`, and `tools` commands.

## 🎯 Why
This fulfills the ROADMAP.md goal to "Add practical examples to command help for analyze, diff, context, gate, cockpit, handoff, run, and export." Some commands had examples, but these were missing them, causing inconsistent documentation.

## 🔎 Evidence
- `docs/ROADMAP.md` states the need for these CLI examples.
- `docs/tutorial.md` and `docs/reference-cli.md` already contain practical examples that can be surface in the CLI help directly.

## 🧭 Options considered
### Option A (recommended)
Add practical `after_help` examples to the CLI command parsers for `diff`, `gate`, `run`, `export`, and `tools` (aligning with `analyze`, `context`, and others which already have them). Then regenerate `docs/reference-cli.md` with `cargo xtask docs --update`.
- Fits because it fulfills the ROADMAP goal and is generated automatically.
- Trade-offs: Structure/Velocity - Adds boilerplate to parser files but significantly improves developer experience.

### Option B
Do not modify the CLI parsers, but only append examples to `docs/reference-cli.md` manually.
- Choose when the help text is meant to be hand-maintained separately from the code.
- Trade-offs: Bad governance. The CLI docs are generated by `cargo xtask docs`, so manual edits will be overwritten.

## ✅ Decision
Option A. Added the `#[command(after_help = "...")]` attributes to the CLI arg structs and updated the docs via `cargo xtask docs --update`.
