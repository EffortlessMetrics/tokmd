## Findings
In `context_pack.rs`, the string parser `parse_budget` was widely invoked inside assertions and test cases using `.expect("should exist")`. This message provides zero context if it ever fails. Since Palette focuses on DX, improving error messages to actually describe the failure state is a core objective.

## Options Considered

### Option A (recommended)
- Replace `.expect("should exist")` with `.expect("budget string could not be parsed as a valid token limit")` or similar context-aware messages depending on the code block.
- **Trade-offs:** Minimal blast radius. Only affects test panics and assertions, keeping prod behavior the same but vastly improving test-failure DX.

### Option B
- Refactor the tests to return `anyhow::Result<()>` and use `?` to bubble errors instead of panicking.
- **Trade-offs:** Larger diff, may require more structural changes to test bodies. In tests, `expect` with a good message is often preferred for pinpointing the exact failure line.

## Decision
Option A. It's atomic, safe, and directly improves DX for developers diagnosing failures.
