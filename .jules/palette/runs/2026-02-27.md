# Run Log: 2026-02-27

## Context
- Read: CI workflows, CLAUDE.md, CONTRIBUTING.md
- Lane: Scout Discovery (Lane B)
- Target: CLI Help / Error Message Quality

## Findings
- `tokmd lang` works but has sparse help for complex flags.
- `tokmd context` fails with a helpful message for bad budget values.
- `tokmd module` works as expected.
- `tokmd init`:
    - The `--help` output lists `--template` values (default, rust, node, etc.).
    - However, `tokmd init` (no args) defaults to creating `.tokeignore` silently if non-interactive, or entering a wizard.
    - In non-interactive environments (like this agent sandbox), it just prints "Wrote ./.tokeignore" using the default template.
    - **Friction**: The user might want a specific template (e.g., `rust`) but `tokmd init` without args just picks `default`. The help text shows the default is `default`, but doesn't explain *what* that contains or how to inspect it without writing.
    - **Friction**: `tokmd init --print` is useful but sparse.
    - **Opportunity**: `tokmd init` could be more friendly by explaining what it did, or hinting at available templates if the user didn't specify one.

## Selected Target
**Improve `tokmd init` UX.**
Current behavior:
```
$ tokmd init
Wrote ./.tokeignore
```
Desired behavior:
- Inform the user *which* template was used.
- Hint that other templates exist (if default was used implicitly).
- Maybe suggested next steps (e.g., "Run `tokmd` to see the results").

## Options Considered

### Option A: Enhance `tokmd init` CLI output
- Modify `crates/tokmd/src/commands/init.rs` to print more context.
- "Initialized .tokeignore using 'default' template."
- "Hint: Use --template <NAME> to choose a specific language (rust, node, python...)."
- "Run 'tokmd' to start scanning."

### Option B: Add a `tokmd init --list` or `--show`
- Add a way to see what templates are available without running help.
- Overkill for now; help text already lists values.

## Decision
**Option A**. It's a low-risk, high-value change to "onboarding" UX.
The `init` command is often the first touchpoint. Making it chatty and helpful builds trust.

## Plan
1.  Verify current output of `tokmd init` (already done: `Wrote ./.tokeignore`).
2.  Modify `crates/tokmd/src/commands/init.rs` to:
    - Capture the template name used.
    - Print a friendlier success message.
    - Print a hint about other templates if `default` was used.
3.  Verify with `cargo run -- init`.
