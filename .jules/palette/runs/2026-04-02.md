**Lane selected:** scout
**Target:** Cleanup unwrap/expect in BDD export tests (crates/tokmd/tests/bdd_export_scenarios_w50.rs) to provide better error diagnostics.

**Options considered:**
### Option A
- Replace generic `.unwrap()` and `.expect("failed to execute tokmd...")` with more descriptive `.expect()` messages that indicate exactly what failed (e.g., "tokmd export JSONL should succeed" or "JSON deserialization of metadata row should succeed").
- Refactor test bodies returning Results with `?` to burn down panics where practical, though since this is test code, using descriptive `expect` messages is idiomatic enough and easier to migrate. We will use descriptive `expect()` over `unwrap()`.

### Option B
- Rewrite all tests to return `anyhow::Result<()>` and use `?`.
- This is much more invasive and risks breaking test harnesses if they don't support Result-returning tests natively or output formatting changes.

**Decision:**
Choose **Option A**. It's less invasive, immediately improves developer experience when tests fail (showing exact context rather than a generic line number), and keeps the test signature standard.
