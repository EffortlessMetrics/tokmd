**Target**: Error message clarity when `--explain` is given an unknown metric key.

## Findings
When running `tokmd analyze --explain invalid-key`, the CLI correctly rejects it but gives an error message and a "Hint" that are slightly repetitive:
```
Error: Unknown metric/finding key 'invalid-key'. Use --explain list to see supported keys.

Hints:
- Run `tokmd analyze --explain list` to see supported keys.
```
While not a huge bug, this is a minor DX friction point. I can modify `error_hints.rs` or `analyze.rs` to streamline this and ensure we don't duplicate the same message between the main error and the hint.

## Options considered
### Option A (Streamline error message)
- Remove "Use --explain list..." from `analyze.rs`, letting `error_hints.rs` provide the actionable hint.
- Clean separation of error vs hint.

### Option B (Keep error, remove hint)
- Remove the hint from `error_hints.rs` and keep the long error.
- Less consistent with other errors.

**Decision**: Option A. It's the standard pattern in tokmd to have a short error + clear hint.
