# Palette Run: DX / UX Improvement
## Context
- Read CI gates (cargo fmt, clippy, build, test).
- Lane: scout
- Target: Replace `unwrap()` with explicit `expect()` in `crates/tokmd/tests/determinism.rs` to improve test diagnostics.

## Findings
`crates/tokmd/tests/determinism.rs` contains multiple blind `unwrap()` calls on JSON conversions (e.g. `as_u64().unwrap()`, `as_str().unwrap()`). Replacing these with `.expect("missing <field>")` improves Developer Experience (DX) by making test failures self-explanatory when JSON schemas mismatch or are structurally incomplete.

## Options considered
### Option A
- Replace blind `.unwrap()` in JSON extractions in `crates/tokmd/tests/determinism.rs` with `.expect("<message>")`.
- Provides context on the exact field/invariant that failed (e.g., "code total must exist and be u64").
- High SNR and fits perfectly within the mandate.

### Option B
- Modify global schemas and return `Result` in test signatures instead.
- Pushes failure down but makes test signatures more complex, obscuring where exactly parsing fails.

## Decision
**Option A** is the best choice to quickly burn down blind unwraps into context-rich expects, fulfilling the `Palette` persona's mandate.
