**New Findings:**

Wait, let's add an explicit check for `haystack.contains("worktree")` and `haystack.contains("failed")` or `haystack.contains("invalid reference")` and add a hint.

**Options Considered:**
### Option A (recommended)
- Add a specific hint for `invalid reference` or `git worktree add failed` or `failed to load diff source` which explains: "If you meant to compare files, ensure they exist locally. Otherwise, ensure the git branch, tag, or commit exists."
- Why it fits: Helps users understand why `tokmd diff doesnt_exist doesnt_exist2` gives a weird git error when they might have just mistyped a file path.

### Option B
- Change `diff.rs` to validate if a string looks like a path before passing to git.
- Trade-offs: Might interfere with git ref names that look like paths (e.g. branch names with slashes).

**Decision:** Option A. It's a non-invasive DX win that adds helpful context directly via the existing hints system.
