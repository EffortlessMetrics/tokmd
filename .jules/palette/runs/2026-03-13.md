## Selection
Lane: scout
Target: Refuse to write binary format outputs (e.g. MIDI) to stdout when running in a terminal, preventing garbage characters from breaking the user's terminal session.

## Findings
When users run `tokmd analyze --format midi` (or other binary formats) without redirecting output to a file or specifying an `--output-dir`, `write_analysis_stdout` dumps binary directly to `std::io::stdout()`. This breaks the terminal.
By adding a check using `std::io::IsTerminal`, we can return a friendly error message using `anyhow::bail!` instead.

## Options Considered
**Option A:** Add `is_terminal()` check in `write_analysis_stdout` returning a friendly error if true.
**Option B:** Fallback to a text format if stdout is a terminal. (Could be confusing if they specifically requested MIDI).

**Decision:** Option A is selected as it strictly respects the user's intent while preventing UX damage.
