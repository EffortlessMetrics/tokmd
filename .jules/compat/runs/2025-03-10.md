# Compat Run: 2025-03-10

## Target
`tokmd-python` build fails when `cargo test --no-default-features -p tokmd-python` is run due to linking issues with PyO3.

## Decision
Option A: Split the `extension-module` flag out into a dedicated crate feature which is enabled by default. This satisfies PyO3 docs and prevents cargo test from trying to build the library as a standalone cdylib during tests, causing linking issues.
Option B: Disable testing on the python crate entirely or pass custom rustflags on the test suite.

**Chose Option A** because it is idiomatic to PyO3's Cargo features guide and prevents the need for wrapper scripts or custom cargo commands.
