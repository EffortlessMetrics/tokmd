# Compat Run 2026-04-04

**Target**: `tokmd-progress` fails `clippy` on `--no-default-features` due to `drop_non_drop`.

## Context
When building without default features (which excludes the `ui` feature), the `Progress` and `ProgressBarWithEta` types are compiled as empty mock structs that do not implement `Drop`. Explicitly calling `drop()` on them inside tests triggers `clippy::drop_non_drop`, which fails the strict CI checks (`-D warnings`).

## Options
- **Option A**: Replace `drop(p)` with `let _ = p` in the test files to explicitly discard the value without invoking drop mechanics or tripping the lint.
- **Option B**: Add `#![allow(clippy::drop_non_drop)]` to the top of affected test files.

**Decision**: Chosen Option A because it is more idiomatic and avoids adding lint exceptions.

## Verification
- `cargo clippy -p tokmd-progress --no-default-features --tests -- -D warnings` (PASS)
- `cargo xtask gate` (PASS)
