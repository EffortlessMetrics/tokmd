# Librarian Run $(date +%Y-%m-%d)

## Options Considered

### Option A: Add doctests for `tokmd cockpit`
- **What it is**: Add `tokmd cockpit` usage verification tests to `crates/tokmd/tests/docs.rs` based on the CLI recipes in `docs/recipes.md` and `docs/tutorial.md`.
- **Why it fits this repo**: `tokmd cockpit` is a core review command, but its examples are not verified in `tests/docs.rs`, creating a risk of silent drift.
- **Trade-offs**: Requires setting `--base` explicitly in the test to ensure it runs deterministically without network calls.

### Option B: Add doctests for `tokmd context`
- **What it is**: Add `tokmd context` usage verification tests to `crates/tokmd/tests/docs.rs` based on the CLI recipes in `docs/recipes.md` and `docs/reference-cli.md`.
- **When to choose it instead**: If `tokmd cockpit` requires a fully functional git setup that is hard to mock in a simple CLI test.
- **Trade-offs**: Testing `--mode bundle` requires file output checks, but is straightforward.

## Decision
We'll proceed with **Option B** (Add doctests for `tokmd context`). The `tokmd context` command is highlighted heavily in the README ("Browser-safe context bundles", "pack code for an LLM budget"), but isn't tested in `docs.rs`. `tokmd cockpit` requires git-diffable history between `main` and `HEAD` which is tricky to guarantee in arbitrary shallow clones or sandbox environments, whereas `tokmd context` just operates on the local filesystem.
