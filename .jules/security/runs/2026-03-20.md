## Findings
Multiple unwrap() calls in test functions across tokmd CLI modules:
- `interactive/wizard.rs`
- `commands/baseline.rs`
- `commands/check_ignore.rs`

Using `unwrap()` in tests can cause panics that immediately kill the test runner thread without graceful error reporting.

## Options
### Option A (recommended)
- Change test signatures to return `anyhow::Result<()>` or `Result<(), Box<dyn std::error::Error>>`.
- Use the `?` operator to propagate errors gracefully instead of panicking.
- Fits the repo style of burning down unwraps completely.

### Option B
- Just use `expect("msg")` everywhere.
- Less clean than Option A and doesn't fully remove panicking paths.

## Decision
Option A. It aligns with the repo's strict quality gate requirement to burn down panics, even in tests, and ensures test failures are handled nicely.
