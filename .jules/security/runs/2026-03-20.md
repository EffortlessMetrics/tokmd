# Run 2026-03-20

- **Lane:** Scout
- **Target:** Burn down test unwraps in `tokmd` CLI tests:
  `interactive/wizard.rs`, `commands/baseline.rs`,
  `commands/check_ignore.rs`, and `export_bundle.rs`

## Findings

Multiple `unwrap()` calls in `tokmd` CLI test functions were still
panicking on error paths instead of propagating failures cleanly.

## Options

### Option A (recommended)

- Change test signatures to return `anyhow::Result<()>`.
- Use `?` to propagate setup and decode failures instead of panicking.

### Option B

- Replace `unwrap()` with `expect(...)`.
- Keeps explicit panic paths and does not actually reduce panic-driven test failures.

## Decision

Option A. It matches the repo's panic burn-down policy and produces
better failing-test diagnostics.

## Changes Made

- Updated `crates/tokmd/src/interactive/wizard.rs` tests to return
  `anyhow::Result<()>` and use `?`.
- Updated `crates/tokmd/src/commands/baseline.rs` tests to avoid
  panic-driven option handling.
- Updated `crates/tokmd/src/commands/check_ignore.rs` tests to return
  `anyhow::Result<()>` and use `?`.
- Updated `crates/tokmd/src/export_bundle.rs` tests to return
  `anyhow::Result<()>` and use `?`.
