# Security Run 2026-04-04

**Lane:** scout
**Target:** crates/tokmd-analysis-fun

## Findings
Discovered widespread use of `.unwrap()` in `crates/tokmd-analysis-fun/tests/properties.rs` and `crates/tokmd-analysis-fun/tests/deep_w66.rs`.
Replacing these with `.expect("...")` specifying the exact invariant violated, conforming to the repo quality standard to burn down panics.

## Options Considered
### Option A (recommended)
Replace `.unwrap()` calls with `.expect("...")` containing specific context about why the eco_label or other values should be present.
- Fits the repo's explicit instruction to contextualize unwrap/expect/panic for better DX and Sentinel security burn down.
- Trade-offs: Increases verbosity slightly, but improves determinism and test failure debugging.

### Option B
Change test functions to return `anyhow::Result<()>` and use `?`.
- Not feasible directly for `proptest!` macros without wrapping logic.
- Trade-offs: Changes test signatures, could conflict with macro expansions.

## Decision
Option A. It is the repo-preferred method for test files using proptest and aligns with the mandate to avoid changing signatures where macros are involved.
