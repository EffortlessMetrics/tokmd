## Findings
Found an `unwrap()` in `Progress::new` when setting the spinner style. This panics if the hardcoded template is invalid.

## Decision
**Option A (Recommended)**: Replace `unwrap()` with `expect("valid template")`.
- **Why**: The template string is hardcoded and statically known. `expect` documents this invariant and provides a better panic message if it somehow fails (e.g. library update changes syntax). Changing the signature to return `Result` is overkill for a static string.
- **Trade-offs**: Still panics, but explicit.

**Option B**: Refactor `Progress::new` to return `Result`.
- **Why**: Eliminates panic.
- **Trade-offs**: Ripples changes to all call sites (many). High effort, low value for a static string.

**Selected**: Option A.
