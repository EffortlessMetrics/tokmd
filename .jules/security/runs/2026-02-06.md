# Run Log: 2026-02-06

**Run ID**: 20260206-1333-10284
**Lane**: Lane B - Scout Discovery
**Target**: Deterministic `TomlConfig` serialization

## Findings
- `TomlConfig` uses `HashMap<String, ViewProfile>` for the `view` field.
- This causes non-deterministic output when serializing configuration (e.g. in `tokmd init` or `wizard`).
- `UserConfig` already uses `BTreeMap` for similar fields.
- Found during Scout Discovery of "Deterministic receipt/schema behavior".

## Decision
- **Option A**: Replace `HashMap` with `BTreeMap` in `TomlConfig`.
- This aligns with `UserConfig` and the architectural principle of deterministic output.
- Low risk, high value for tool stability.

## Plan
1. Add reproduction test case to `crates/tokmd-config/tests/determinism.rs`.
2. Verify failure (or flakiness).

4. Verify pass.
