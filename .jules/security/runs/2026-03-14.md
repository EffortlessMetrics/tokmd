# Sentinel Run - 2026-03-14

**Lane**: scout
**Target**: crates/tokmd-core/src/

## Findings
- Discovered numerous `.unwrap()` and `.expect()` calls in `crates/tokmd-core/src/lib.rs` and `crates/tokmd-core/src/ffi.rs` (totaling over 40 panicking calls).
- Many of these are in tests, which can be easily refactored to return `Result<(), Box<dyn std::error::Error>>` and use the `?` operator.

## Options considered
### Option A (recommended)
- **What it is**: Refactor the tests in `crates/tokmd-core/src/lib.rs` and `crates/tokmd-core/src/ffi.rs` to burn down the `unwrap` and `expect` calls by modifying test signatures to return `Result<(), Box<dyn std::error::Error>>`.
- **Why it fits this repo**: Fits the Sentinel goal of burning down panics ("treat unwrap/expect/panic as candidates everywhere (including tests)"). Improves test reliability.
- **Trade-offs**: Requires modifying many test definitions, but it is a highly localized and mechanical change that provides strong evidence of quality improvement.

### Option B
- **What it is**: Select just one file (e.g., `crates/tokmd-core/src/lib.rs`) to refactor instead of both.
- **When to choose it instead**: If modifying both files makes the PR too large.
- **Trade-offs**: Leaves panics in the other file, reducing the overall impact.

## Decision
Choosing **Option A** because the blast radius is still constrained to tests in `tokmd-core`, and burning down all of them in this core crate is a high-leverage win for repo quality.
