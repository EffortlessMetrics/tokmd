## Findings
Found extensive use of `expect()` in Python bindings (`tokmd-python`) when setting dictionary items or converting arguments. While `PyDict::set_item` rarely fails, panicking in a library (especially via FFI) is poor practice and can crash the host process.

Refactored `crates/tokmd-python/src/lib.rs` to propagate `PyResult` errors using `?` instead of panicking.
