# Security Run: 2026-03-24
Run ID: 36a12d57-d1b7-4a61-8258-dd6dee75b0de

## Target
Lane: scout
Target: Burn down .unwrap() in `crates/tokmd/src/context_pack.rs`

## Findings
We found 31 unwraps in `crates/tokmd/src/context_pack.rs` test suites. This violates the goal of high-quality unwrap burn-down in Rust. They were panic-happy without clear messages.

## Options Considered
### Option A
Refactor the tests to return `anyhow::Result<()>` and use the `?` operator.
This fits the repo since it provides correct error bubbling during test failures and eliminates panics, preventing accidental panicking from hiding the true cause.

### Option B
Use `.expect("msg")` on all `.unwrap()` calls.
This keeps panics but provides context.

## Decision
Chosen Option A for most of the parser unwraps, and Option B (via `.expect("msg")`) where we just need a localized assertion without rewriting iterator pipelines entirely. It's the cleanest combination that enforces tight scope.
