# Sentinel Run 2026-03-25

**Lane Selected:** scout
**Target:** `crates/tokmd/src/export_bundle.rs` and `crates/tokmd/src/commands/sensor.rs` tests unwrap burn-down

## Findings
Tests in `crates/tokmd/src/export_bundle.rs` and `crates/tokmd/src/commands/sensor.rs` were using `.unwrap()`, which is an anti-pattern for resilient tests and violates the Sentinel unwrap burn-down directive.

## Options considered
### Option A (recommended)
- Replace `.unwrap()` with `?` where the function can return a `Result`.
- For assertions on `Option`s, use `.map` with `Some(...)`.
- Trade-offs: Minor test refactoring, safer assertions.

### Option B
- Ignore unwraps in tests.
- Trade-offs: Fails to burn-down all `unwrap`s, leaving potential for hard panics in tests.

## Decision
Option A was chosen as it aligns with the strict code quality enforcement for the repo to burn down `.unwrap()` everywhere.

## Verification
- Clean compilation and `clippy` execution.
- `tokmd` tests successfully passed locally.
