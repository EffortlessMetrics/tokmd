# Security Run: 2026-03-21

**Lane:** Scout
**Target:** crates/tokmd-content/src/complexity.rs

## Findings
The codebase contains several `unwrap()` calls on static regex compilation in `LazyLock` blocks. This violates the 'no unhandled panics' quality rule, which recommends using `expect` with a clear justification for static code, or returning a `Result`.

## Options considered
### Option A (recommended)
- Replace `.unwrap()` with `.expect("Static regex must compile")` in `crates/tokmd-content/src/complexity.rs`.
- Why it fits: Aligns with the burn-down goal for unprotected panics while maintaining performance (static regex).
- Trade-offs: Still a panic, but documented and intentional.

### Option B
- Refactor to return a `Result` or avoid regex entirely.
- When to choose: When regex pattern is dynamic.
- Trade-offs: Too much structural overhead for known static strings.

## ✅ Decision
Option A. It is SRP, tight scope, and directly addresses the panic burn-down goal for static regexes.
