## Target
Reduce unnecessary string allocations in `tokmd-model`'s `collect_file_rows` loop by switching `Key` to use `Cow` and string references instead of owned strings.

## Proof Method
Existing benchmark harness via `cargo bench -p tokmd-model`.

## Options considered
### Option A (recommended)
- What it is: Change `Key` from owning `String`s to borrowing them where possible using `Cow<'a, str>` and `&'a str`.
- Why it fits this repo: Preserves BTreeMap determinism while structurally avoiding allocations during hot-path aggregation loop.
- Trade-offs: Requires small lifetime annotations on internal structs but improves runtime safely.

### Option B
- What it is: Swap BTreeMap for HashMap.
- When to choose it instead: When key ordering doesn't matter.
- Trade-offs: Breaks the project's strict requirement for deterministic JSON and receipt outputs (explicitly stated in `AGENTS.md` and tests).

## ✅ Decision
Option A. Keeps deterministic iteration order intact while bringing measurable wins to aggregation throughput.
