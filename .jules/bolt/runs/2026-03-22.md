# Bolt Run - 20260322T130337Z

- Lane: scout
- Run ID: 3eaaed8a-553d-46e7-aea7-9e7b44e684e9
- Target: hot aggregation loops in `tokmd-model` and `tokmd-analysis-derived`

## Findings

- `tokmd-model` cloned owned language and module keys in hot
  aggregation loops even when the keys already existed in the map.
- `tokmd-analysis-derived` can borrow `lang` and `module` keys
  during ratio and rate grouping and allocate only on insert
  misses.

## Options

### Option A

- Use borrowed-key lookups and allocate only when inserting a new `String` key.
- Keeps the receipt types and public interfaces unchanged.

### Option B

- Push borrowed keys through the wider report types and helper APIs.
- Saves more allocations eventually, but widens the refactor well beyond this lane.

## Decision

Option A. Keep the PR local, make the allocation win real in both
crates, and preserve the existing public types.
