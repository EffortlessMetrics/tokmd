## Target
`crates/tokmd-analysis-content/src/content.rs` - Optimize `build_import_report` by removing unnecessary `String` allocations (specifically `.clone()`) when accumulating imports into the `edges` `BTreeMap`. By changing the map key from `(String, String)` to `(&str, String)`, we reuse the borrowed slice from the source module/file string, preventing redundant memory allocations in the hot loop.

## Options Considered

### Option A (recommended)
- **What it is:** Refactor `edges` to use `(&str, String)` as the key.
- **Why it fits:** Reduces allocations in the critical path without external dependencies.
- **Trade-offs:** Requires an allocation at the end when mapping to `ImportEdge`, but this is performed only once per unique edge, rather than on every iteration of the parsing loop.

### Option B
- **What it is:** Use a custom hasher or `StringCache` to intern strings.
- **Why to choose:** Useful if the number of unique strings is very small.
- **Trade-offs:** Introduces new dependencies and architectural complexity.

## Decision
**Option A** was chosen.
