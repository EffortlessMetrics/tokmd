## Problem
Persona files contain duplicated boilerplate (e.g., the `## Notes` section), which looks like a prime target for deduplication. However, deduplicating them breaks the agent's context because agents only receive their specific persona file.

## Evidence
- `.jules/personas/*/README.md` all have identical `## Notes` sections.
- Attempted to deduplicate in PR, but reviewer rejected it: "prompt-critical guidance must stay in the individual persona files because Jules receives personas individually."

## Why it matters
Future Archivist runs might also see this duplication and try to "fix" it, wasting time and API calls. We need a way to mark intentional duplication so agents ignore it.

## Done when
- [ ] A shared lint rule or AGENTS.md note explicitly protects the `## Notes` boilerplate from deduplication.
