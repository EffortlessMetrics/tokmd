## Problem
In `crates/tokmd-node/Cargo.toml`, the `tokio` dependency is explicitly requested with the `rt-multi-thread` feature. However, the `napi` dependency's `async` feature already automatically pulls this in via `tokio_rt`. The explicit request is redundant and can be safely tightened to `tokio = "1"`.

## Evidence
- `crates/tokmd-node/Cargo.toml`
- Running `cargo tree --workspace --edges no-dev | grep "rt-multi-thread"` points out the redundant pulling.

## Why it matters
It reduces the explicit dependency surface area and cleans up dependency hygiene in the bindings target, which makes future upgrades more predictable.

## Done when
- [ ] The `rt-multi-thread` feature is removed from the `tokio` dependency block in `crates/tokmd-node/Cargo.toml`.
