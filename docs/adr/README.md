# Architecture Decision Records

This folder contains architecture decision records for tokmd.

- [ADR-001: Analysis Facade Boundary in tokmd-core](./ADR-001-analysis-facade-boundary.md)
- [ADR-002: Deterministic Receipt Ordering and Normalization](./ADR-002-deterministic-receipt-ordering.md)

When adding a new ADR:
1. Use `ADR-XXX-short-title.md` naming.
2. Include status and date.
3. Capture context, decision, consequences, and alternatives.
4. Link related docs (architecture, design, schema, or implementation plan).
